Skill 开发
持续 · Creator 花时间最多的阶段
5.1 Skill 结构设计
Plugin 逐步引导 Creator 完成三步法设计：
三步法
1
确定输入 — 用户需要提供什么信息？（如：一个关键词、一组 ASIN）
2
拆解步骤 — 拿到输入后分几步完成？（如：数据采集 → 竞争分析 → 推荐词）
3
定义输出 — 最终给用户看什么结果？（如：推荐词列表 + 评分 + 摘要）
写入结构定义
将输入、步骤、输出通过平台 API 写入后端
自动
逐个节点
每个节点重复
5.2a 数据需求发现
通过 MCP 查询数据服务商可用 API 列表，向 Creator 推荐匹配当前节点的接口
MCP 发现
5.2b 确认数据需求 & 处理逻辑
Creator 选择需要的 API → 确认处理方式（直接透传 / 聚合 / LLM 分析）
5.2c 生成节点代码
三层分离：数据获取（HTTP 直调）→ 数据处理（纯代码或 LLM）→ 输出渲染（minus.output.*）
同时定义该节点的输出和用户确认行为
自动生成
5.2d 开发即测试
Creator 在浏览器访问 localhost:{port}/preview 亲自体验效果，提出修改意见，Plugin 即时调整
手动体验
上下文容量检查
容量充足 → 继续开发下一节点
接近上限 → 保存进度到 Memory，引导开新 Session
所有节点完成
5.3 结果呈现设计
Plugin 引导 Creator 逐一确认最终结果页的四个方面：
四维确认
1
结果数据 — 最终结果包含哪些数据？
2
结果摘要 — LLM 自动生成还是固定模板？
3
内容预览 — 表格、卡片、图表？
4
下载内容 — Excel / HTML 报告？
5.4 生成输出 Schema（v1 预留）
自动根据结果数据生成 output_schema 写入后端，不对 Creator 展示。为后续 Skill 链路对接预留
静默

自动

手动 · 最终确认
